  • Programming Language : JAVA, HTML, JavaScript, CSS, Python, C

담당 파트

  • 파이선을 통한 Open Api 활용과 가공 및 데이터 크롤링
  • 라즈베리파이 카메라 모듈을 이용한 사진 촬영
  • 데이터베이스 설계
  • MongoTemplate를 사용한 데이터베이스로의 접근과 CRUD 연산 구현
  • Ajax를 통한 비동기 처리
  • 식물도감이나 게시판 등의 JScroll을 통한 무한스크롤 구현
  • 게시물 검색 기능
  • 프로젝트의 전체적인 프론트엔드

도담도담은 IoT센서를 사용해 나의 반려식물을 더 잘 키울 수 있게 도와주는 교육용 서비스입니다.

이 아이디어는 최근 아동 및 청소년들의 감수성 저하와 폭력성 증가에 대한 문제로부터 시작하게 되었습니다.
이러한 문제점들을 해결하기 위한 방법을 조사하던 중 '식물 체험 활동이 아동의 정서 발달에 긍정적인 효과를 미친다'는 연구 결과를 찾게 되었습니다.
따라서 저희는, 아이들이 식물을 기르는 것에 대해 더 관심을 가지도록 하여, 식물이 주는 긍정적인 영향을 받을 수 있게 해주기 위한 서비스를 기획하게 되었습니다.

웹은 자바 스프링 프레임워크를 사용하여 제작하였고 데이터는 센서에서 받게되는 대량의 데이터를 저장하기 위해 NoSQL인 MongoDB를 사용했습니다.

전체적인 데이터의 흐름은 다음과 같습니다.

1. 아두이노 NodeMCU에 부착된 온습도 센서, 토양 수분 센서는 일정 간격으로 화분의 상태를 측정합니다.
2. 측정된 데이터를 Hive MQTT를 이용해 Node.JS 서버로 전달합니다.
3. Node.JS 서버는 구독해온 화분 데이터를 MongoDB에 저장합니다.
4. 사용자는 제공되는 스마트폰 앱을 통해 하드웨어 기기를 등록합니다.
5. 등록된 기기를 통해 측정된 데이터는 스마트폰과 PC환경에서 조회가 가능합니다.

부착된 센서에서 값이 측정되는 모습입니다.
이렇게 측정된 데이터는 MQTT 브로커를 통해 발행되고 Node.JS는 이를 구독해 MongoDB에 Insert합니다.

스마트폰 앱에서 해당 기기를 블루투스로 검색해 와이파이를 연결해줍니다.
등록이 완료되면 웹에서 해당 기기가 부착된 화분을 등록할 수 있고 애칭이나 식물 종류를 정할 수 있습니다.

도담도담 웹에서 기기가 부착된 화분의 애칭과 식물 종을 입력합니다.
이렇게 반려식물의 정보가 등록되면 웹에서 화분의 상태에 대한 다양한 정보가 제공됩니다.

등록된 화분의 온습도 정보 탭에 들어가면 센서를 통해 측정된 데이터를 기반으로 화분의 상태를 확인할 수 있습니다.

우선, 식물 양육에 대한 기본적인 가이드라인이 제시됩니다. 식물 종에 대한 물을 주는 주기와, 채광 및 습도 정보가 제공됩니다.
가이드라인에 대한 데이터는 식물에 대한 정보를 제공하는 기존의 웹사이트들을 파이썬을 이용해 크롤링해 데이터를 수집하였습니다.
수집한 데이터는 파싱하여 데이터베이스에 저장하였고, 사용자의 식물 종에 맞추어 제공해줍니다.

센서를 통해 측정된 데이터는 구글 차트 api를 사용해 시각화해주었습니다.
온도, 습도 그리고 토양 수분 이렇게 세가지 데이터를 시각적으로 표현함으로서 사용자가 식물의 상태를 더 객관적으로 알수 있게 하였습니다.

추가적으로, 센서를 통해 측정된 데이터의 평균값을 측정하고 같은 식물을 키우는 사람들과 식물 상태를 비교해볼 수도 있습니다.
다른 사람들의 데이터에와 비교해 수치가 상당히 높을 경우 붉은색, 낮을 경우에는 푸른색을 띄도록 구현하였습니다.
물론, 전체의 평균이 식물에게 최적의 상태라고 하긴 힘들지만, 식물을 처음 키우는 사람들에게는 좋은 가이드 라인이 될 수 있다고 생각합니다.

물주기 달력 탭에서는 사용자가 식물에게 물을 언제 줬는지 확인할 수 있습니다.
물을 줬는지 여부는 토양 수분 센서가 수치가 급격하게 증가한 경우를 Node.js서버에서 체크해 자동으로 달력에 표시되도록 구현했습니다.
따라서, 사용자가 직접 물을 준 날짜를 기록하지 않더라도 물을 준 날짜를 쉽게 확인할 수 있습니다.

해당 화분의 성장과정 탭에서는 라즈베리파이 카메라 모듈로 촬영된 화분의 사진을 확인할 수 있습니다.
라즈베리파이는 매일 하루에 한장씩 지정된 시간에 화분의 모습을 촬영하여 웹 서버로 전송합니다.
웹에서 사용자는 달력에서 기간을 선택하면 해당 기간 동안의 식물의 모습을 확인할 수 있습니다.
따라서 반려식물의 성장 과정이 매일 사진으로 기록되고 사용자는 이를 참고하여 식물의 상태를 파악할 수 있습니다.

식물도감 탭에서는 다양한 식물에 관한 정보를 볼 수 있습니다.
약 400여 종 이상의 식물이 등록되어있으며, 식물의 양육 정보, 꽃말이나 특성 등에 대한 정보를 제공합니다.
식물 데이터는 기본적으로 농촌진흥청에서 제공하는 Open Api를 사용했습니다. xml형식의 데이터를 파이썬을 이용해 파싱하여 데이터베이스에 저장해주었습니다.
추가적으로 다양한 식물 정보 사이트의 데이터를 크롤링해와 식물도감의 데이터로 사용하였습니다.

사용자는 관찰일지 탭에서 해당 식물에 대한 관찰 내용과 사진을 등록할 수 있습니다.
그리고 관찰일지를 작성하게되면, 작성된 시점으로부터 가장 최근에 측정된 센서의 데이터가 함께 기록됩니다.
따라서 사용자는 식물의 특이사항이나 성장과정을 더욱 객관적으로 기록할 수 있습니다.

댓글과 좋아요가 가능한 간단한 자유게시판을 구현하였습니다.
이 공간에서 사용자는 식물과 관련해 자유롭게 소통할 수 있습니다.

모바일 앱에서도 마찬가지로 등록된 화분의 정보를 확인할 수 있습니다.
추가적으로 앱에서는 사용자에게 재미적인 요소를 충분히 더 많이 줄 수 있게끔 다양한 기능들이 구현되었습니다.

안드로이드 스튜디오로 앱을 구현할 때 데이터베이스인 몽고DB 아틀라스에 직접 접근하는것이 불가능하여 Spring에서 앱 동작만을 위한 컨트롤러를 추가로 구현해주었습니다.
안드로이드 스튜디오에서 데이터를 통신할 때 Handler라는 곳에 필요한 데이터를 담아 http 통신을 이용해 웹 서버로 전송합니다.
컨트롤러에서는 해당 데이터를 전달받아 필요한 로직을 동작하고 그 결과를 다시 안드로이드 스튜디오로 리턴해줍니다.
이러한 방식으로 모바일 앱과 데이터베이스 사이의 연결 환경을 구현하였습니다.

  • O/S : Windows 10(개발환경)
  • Server: Tomcat v9.0
  • DB: Oracle 11g
  • Framework/flatform : Spring
  • Programming Language : JAVA, HTML
  • Tool : Eclipse, SQL Developer
  • API : KaKao Developer

담당 파트

  • 웹사이트 프론트엔드 코드 크롤링
  • Oracle JDBC를 통한 DB연동
  • 회원가입 및 로그인 구현
  • 파일 업로드가 가능한 게시판 구현
  • 회원 및 매장 관리 구현
  • 피자 판매량 통계 구현

 이 프로젝트는 모델2 기반의 JSP 웹 프로그래밍으로 도미노피자 웹 사이트를 재구현해 본 프로젝트입니다.
JDBC와 MVC 패턴의 데이터 흐름 등을 직접 경험해보기 위한 백엔드 위주의 프로젝트로,
프론트엔드는 실제 도미노피자 웹 사이트를 크롤링하여 개발하였습니다.

데이터베이스는 관계형 데이터베이스 Oracle을 사용하였습니다. 회원으로 로그인하면 피자를 주문 할 수 있도록 회원 테이블과 주문내역 그리고 메뉴 위주로 테이블을 구성하였습니다.
추가적으로 웹 사이트의 공지사항이나 이벤트와 같은 게시판을 구현하기 위해 게시판 테이블도 생성해주었습니다.

관리자 계정으로 접속하면 회원 관리, 매장 관리, 메뉴 관리, 관리자 게시판, 판매량 통계의 다섯가지 탭이 존재합니다.
회원 관리 탭에서는 웹 사이트에 회원 가입한 회원들의 정보를 확인하고 관리 할 수 있습니다.

매장관리 탭에서도 마찬가지로 등록된 매장에 대한 정보를 확인할 수 있고 새로운 매장 정보를 등록할 수 있습니다.

이러한 서비스들은 해당 기능을 담당하는 Action Class를 생성해 데이터베이스에 접근하여 기본적인 CRUD 연산이 가능하도록 구현하였습니다.
각각의 Action Class들은 DAO에 접근해서 회원 및 매장 테이블에 데이터를 수정하거나 삭제, 생성하게됩니다.

관리자는 메뉴관리 탭에서 피자나 사이드메뉴 혹은 음료수와 같은 메뉴들을 등록할 수 있습니다.
이렇게 등록된 메뉴는 데이터베이스의 메뉴 테이블에 저장되게 되고 홈페이지에서 주문 가능합니다.

관리자 페이지에서 등록한 메뉴들이 실제 메뉴로 등록된 모습입니다.
원하는 메뉴를 클릭하면 메뉴를 주문할 수 있는 화면으로 이동하게됩니다.

피자의 사이즈와 도우 그리고 토핑 등을 선택하고 주문하기를 클릭하면 선택한 메뉴들이 장바구니에 담기게됩니다.
장바구니의 제품들은 Session Scope에 저장되어있으며 추가로 다른 제품을 장바구니에 담게되면 이 Session Scope에 제품이 계속 저장됩니다.
그리고 주문하기 버튼을 클릭하면 이 항목들이 주문 내역 테이블에 저장됩니다.

주문내역에 저장된 메뉴들은 관리자페이지의 판매통계 탭에서 판매량 확인이 가능합니다.
메뉴별 판매량이 월별로 표시되며, 판매량 상위 세 개의 피자를 띄워줌으로서 인기 피자 메뉴를 알 수 있습니다.
